# 电机试点最终目标记录

## 已确认的业务规则

- 采购日期、批次号、供应商、品牌不进入当前电机试点录入流程。
- 照片只用于方便显示和列表查找，不作为每次入库、出库的必填资料。
- 只有第一次购入并建档的电机需要生成唯一 SN/内部编码，并基于这个编码生成二维码或条形码。
- 编码格式为 `型号-递增编号`，例如 `GM6020-0001`。
- 每种型号单独递增编号。
- 已经使用过的编号不复用，即使电机报废或作废，也继续往后编号。
- 入库流程应以扫码为主：扫到电机编码后直接入库，不再要求二次拍照和填写复杂字段。
- 出库流程应以扫码为主：扫到电机编码后，只登记“谁出的”和“给哪个车使用”。
- 列表页需要能通过照片缩略图、编码、型号、状态快速找到电机。

## 完整 demo 闭环

下一版必须先做完整 demo，再投入现场测试。demo 应呈现最终设想的完整工作方式：

1. 电脑端新建电机档案。
2. 系统生成 `GM6020-0001` 这类编码。
3. 电脑端生成二维码和 Code128。
4. 电脑端进入标签打印页，打印或复制编码到本地打标软件。
5. 标签贴到电机或收纳位置。
6. 手机端扫码或输入编码查询电机。
7. 手机端扫码或输入编码完成入库。
8. 手机端扫码或输入编码完成出库，并填写出库人和车辆。
9. 电脑端列表、详情和日志能看到完整状态变化。

## 当前实现取舍

- 现阶段使用 `型号-递增编号` 作为系统生成的唯一编码，并同步写入 SN 字段。
- 二维码和 Code128 当前展示的都是该唯一编码。
- 数据库中暂时保留品牌、供应商、批次、采购日期等旧字段，避免当前阶段做破坏性迁移；界面和业务流程不再使用这些字段。
- 当前扫码以输入框承接扫码枪输出为基础；手机端先保留手动输入降级，摄像头扫码作为可替换能力继续接入。
- 当前打标机不能联网，系统不做直连打印，只提供标签页面、二维码、条形码和复制编码能力。
