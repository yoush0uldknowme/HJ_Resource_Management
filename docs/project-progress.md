# 电机资源管理系统进度表

本文档用于记录项目整体计划、当前进度和后续待办。每完成一部分，就在这里更新状态，避免需求和实现过程混乱。

## 当前阶段

当前项目处于：**完整 demo 闭环开发阶段**。

目标不是先做一部分就投入现场测试，而是先做出一版能完整呈现预期工作方式的 demo：

`电脑端建档 -> 自动生成唯一编码 -> 标签打印 -> 手机端扫码查询 -> 手机端扫码入库 -> 手机端扫码出库 -> 电脑端查看状态和日志`

当前试用地址：

`http://localhost:4011/login`

默认账号：

- 管理员：`admin / admin123`
- 查看账号：`viewer / viewer123`

## 总体目标

先完成电机管理 demo，不扩展完整资源管理平台。第一阶段验证电机管理流程是否真的能在现场跑通。

核心目标：

- 每台电机首次购入时建档
- 系统自动生成唯一内部编码
- 基于唯一编码生成二维码和 Code128 条形码
- 照片仅用于列表和详情页辅助识别
- 电脑端用于建档、查看、标签打印和追踪记录
- 手机端用于现场扫码查询、入库和出库
- 所有关键操作保留历史记录

## 当前明确不做

- 不做微信小程序
- 不做打标机直连
- 不做完整权限体系
- 不做电机编辑、删除、作废
- 不做数据导出和自动备份
- 不扩展裁判系统、打印耗材或其他物资模块

## demo 完成后的现场测试重点

- 标签尺寸是否合适
- 手机扫码成功率是否足够
- 入库和出库步骤是否顺手
- 出库人和车辆字段是否够用
- 列表照片是否能帮助识别电机
- 是否需要增加编辑、作废、导出或备份能力
